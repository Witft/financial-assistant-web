# 财务助手需求规格说明书

## 1. 项目概述

| 项目 | 内容 |
|------|------|
| 项目名称 | 个人财务规划助手 |
| 项目路径 | `/code_space/financial-assistant-web/` |
| 用户 | meng（个人使用） |
| 数据源 | 支付宝 + 微信账单（CSV格式） |
| 核心价值 | 统一分析双账户、区分家庭支持与个人支出、可视化 + AI 建议 |

---

## 2. 用户故事（User Stories）

```
作为用户，我想...
1. 上传支付宝/微信账单文件，自动解析并合并分析
2. 看到本月收入/支出/结余的总体概况
3. 查看支出分类占比（餐饮/交通/购物...）
4. 区分"个人消费"和"给家里的转账"
5. 看到最近N笔交易记录
6. 获取AI提供的财务建议
7. 查看历史月份的数据对比
```

## 7. 交互流程（功能1：账单上传）

| 步骤 | 描述 |
|------|------|
| 1 | 首页展示上传区域（拖拽 + 点击） |
| 2 | 用户选择/拖入 CSV 文件 |
| 3 | 前端自动识别来源（支付宝/微信） |
| 4 | 解析成统一格式的交易数据 |
| 5 | 存入 localStorage（覆盖策略） |
| 6 | 自动跳转到 Dashboard |

### 自动识别规则
- 支付宝：CSV 包含 `交易号`, `商品说明` 列，GBK 编码
- 微信：CSV 包含 `交易时间`, `交易类型` 列，UTF-8 编码

### 数据存储
- 策略：覆盖（每次上传替换已有数据）
- 存储位置：`localStorage`

## 8. 技术架构（解析模块）

```
src/
├── utils/
│   ├── parsers/
│   │   ├── index.js        # 统一入口，自动识别调用
│   │   ├── alipay.js       # 支付宝解析器
│   │   └── wechat.js       # 微信解析器
│   └── storage.js          # localStorage 封装
```

### 统一数据结构（Transaction）

| 字段 | 类型 | 描述 |
|------|------|------|
| id | string | 唯一ID |
| date | string | ISO 日期 |
| amount | number | 金额（收入+，支出-） |
| category | string | 分类 |
| description | string | 描述 |
| source | 'alipay' \| 'wechat' | 来源 |
| type | 'income' \| 'expense' \| 'transfer' | 类型 |
